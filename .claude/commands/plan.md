# Plan

Before writing any code:

1. **Restate** what needs to be built in 2-3 sentences
2. **Identify files** that will be created or modified
3. **Break down** into small, sequential steps (max 5)
4. **Flag risks**: breaking changes, large file modifications (especially routes.py), missing dependencies, security concerns
5. **WAIT** for explicit user confirmation before proceeding

Keep the plan concise. No time estimates. No code in the plan.

If the user says "modify" or gives feedback, revise the plan and wait again.
